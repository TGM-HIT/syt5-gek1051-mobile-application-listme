# ID-12 — Teilnehmer entfernen

**User Story:** Als Benutzer möchte ich Teilnehmer aus einer geteilten Liste entfernen können, um die Zugriffskontrolle zu wahren.

**Priorität:** Should Have | **SP:** 3 | **HEAD:** SIEL

---

## Overview

The owner of a shared list can remove other participants. Removed devices lose access immediately — subsequent API calls return 403. The participant list updates in real time for all remaining members via WebSocket.

---

## Backend

- `ListDeviceRepository` — `deleteByListIdAndDeviceId(listId, deviceId)`
- `ParticipantController` — `DELETE /api/lists/{listId}/participants/{deviceId}`
  - Validates that the requesting device is the list owner
  - Deletes the `list_devices` row
  - Broadcasts a `PARTICIPANT_REMOVED` event over WebSocket to `/topic/list/{listId}`

---

## Frontend

- `src/components/list/ParticipantList.vue` — lists all participants with a remove button (owner only)
- `src/components/list/ParticipantSheet.vue` — bottom sheet triggered from list detail header
- On `PARTICIPANT_REMOVED` WebSocket event: removes the device from the presence store and hides the list for that device

---

## Manual Verification

1. Share a list with Device B.
2. On Device A (owner), open participants → tap remove next to Device B.
3. Verify Device B's participant entry disappears from A's list.
4. Verify Device B can no longer fetch items for that list (403).
