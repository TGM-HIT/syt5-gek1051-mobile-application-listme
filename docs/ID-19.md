# ID-19 — Voice Input (Spracheingabe)

Issue: ID 19

Files to know:
- Frontend component: `listme-frontend/src/components/item/VoiceInput.vue`
- Frontend integration: `listme-frontend/src/components/item/AddItemSheet.vue`
- Test: `listme-frontend/src/components/item/VoiceInput.spec.ts`

---

## Overview

Users can add items by speaking instead of typing. A microphone button in the `AddItemSheet` activates the browser's Web Speech API (German, `de-DE`). The recognised transcript is parsed for quantity, unit, and price before filling the form fields automatically.

---

## Frontend

### VoiceInput.vue

A single button that toggles speech recognition on/off.

**Props:**

| Prop   | Type             | Default | Description          |
| ------ | ---------------- | ------- | -------------------- |
| `size` | `'sm' \| 'md'`  | `'sm'`  | Button size (36/40px)|

**Emits:**

| Event    | Payload        | Description                                 |
| -------- | -------------- | ------------------------------------------- |
| `result` | `text: string` | Transcript after recognition ends           |
| `error`  | `msg: string`  | Emitted on `onerror` with a German message  |

**Behaviour:**

1. On first render, checks for `SpeechRecognition` / `webkitSpeechRecognition` on `window`. If absent the button is disabled and greyed out.
2. Tap → `start()`: creates a `SpeechRecognition` instance (`lang: 'de-DE'`, `interimResults: false`, `maxAlternatives: 1`), calls `start()`, sets `isListening = true`.
3. While listening, the button turns red and a pulse ring animates around it (`animate-ping`).
4. `onresult` fires → emits `result` with `e.results[0][0].transcript`.
5. `onend` fires → resets `isListening = false`.
6. `onerror` fires → emits `error('Spracheingabe fehlgeschlagen')`, resets `isListening`.
7. Tap again while listening → calls `recognition.stop()`.
8. `onUnmounted` hook calls `stop()` to release the microphone if the component is destroyed mid-recording.

**Browser support:** Chrome and Chromium-based browsers. Firefox does not implement the Web Speech API and will see the disabled state.

### AddItemSheet.vue integration

`VoiceInput` sits in the input row alongside the barcode and library buttons (visible only when not in edit mode).

**`onVoiceResult(transcript)`** receives the raw transcript and calls `parseVoice()`:

**`parseVoice(raw)` — parsing pipeline:**

| Step | Pattern                                      | Example input           | Result                              |
| ---- | -------------------------------------------- | ----------------------- | ----------------------------------- |
| Pre  | German number words → digits (`toDigits()`)  | `"zwei kg Mehl"`        | `"2 kg Mehl"`                       |
| 1    | Price extraction: `(\d+[.,]\d+?)\s*(?:€\|euro\|eur)` | `"Mehl für 1,99 Euro"` | `price=1.99`, text becomes `"Mehl"` |
| 2    | `[qty] [unit] [name]`                        | `"2 kg Mehl"`           | `qty=2, unit='kg', name='Mehl'`     |
| 3    | `[name] [qty] [unit]`                        | `"Mehl 2 kg"`           | `qty=2, unit='kg', name='Mehl'`     |
| 4    | `[name] [qty]` (trailing number)             | `"Äpfel 3"`             | `qty=3, name='Äpfel'`               |

Supported unit aliases (normalised via `UNIT_MAP`): `kg`, `g`, `l`/`liter`, `ml`, `stk`/`stück`, `pkg`/`packung`, `fl`/`flasche`, `dose`, `bund`, `pck`, `tl`, `el`.

After parsing, the handler fills `name`, `quantity`, `quantityUnit`, and `price` in the form and re-focuses the name input for corrections.

## No backend changes

Voice input is fully client-side. The filled form is submitted through the normal `POST /api/lists/{listId}/items` flow (see [ID-04.5](ID-04.5-Create-Item.md)).

---

## Edge cases

- **Unsupported browser:** Button disabled with tooltip "Spracheingabe nicht verfügbar"; no crash.
- **Microphone permission denied:** `onerror` fires → emits `error`, button returns to idle state.
- **Ambiguous transcript:** If no pattern matches, the full transcript lands in `name`; `qty`, `unit`, `price` remain empty — user corrects manually.
- **Number words:** `SpeechRecognition` sometimes returns spoken numbers as words ("zwei", "drei"); `toDigits()` normalises these before pattern matching.
- **Component unmounted mid-recording:** `onUnmounted` calls `stop()`, preventing a dangling microphone indicator in the OS.

## Manual verification

1. **Happy path — name only**
   - Open a list → tap the mic button → say "Milch" → expect name field filled with "Milch"

2. **Quantity + unit**
   - Say "2 kg Mehl" → expect name="Mehl", qty=2, unit="kg"

3. **Number words**
   - Say "zwei Liter Milch" → expect name="Milch", qty=2, unit="l"

4. **Price extraction**
   - Say "Käse für 3,50 Euro" → expect name="Käse", price=3.50

5. **Unsupported browser**
   - Open in Firefox → expect mic button disabled/greyed out

6. **Stop mid-recording**
   - Tap mic to start → tap again → expect button returns to idle, no result emitted
