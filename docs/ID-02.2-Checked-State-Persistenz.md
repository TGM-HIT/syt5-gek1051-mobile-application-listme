## Overview

This sub-ticket ensures that the `checked` state of every item survives a page reload. Persistence is achieved in two layers: the IndexedDB write-through cache (`CacheService`) stores every toggled item immediately, and the CRDT `OperationQueue` queues the toggle when the device is offline so it can be replayed against the server on reconnect. On next app start, `fetchAll` serves cached items instantly before the network response arrives.

## API Call

**File:** `src/services/item.ts`

The service issues a `PATCH` to the backend when online. The server toggles the boolean and returns the full updated item — the client never computes the new value itself when online.

```typescript
toggleCheck(listId: string, itemId: string): Promise<Item> {
  return api.patch<Item>(`/lists/${listId}/items/${itemId}/check`).then(r => r.data)
}
```

| Method | Endpoint | Body | Response |
|--------|----------|------|----------|
| `PATCH` | `/lists/:listId/items/:itemId/check` | _(none)_ | Updated `Item` |

## Store Action: `toggleCheck`

**File:** `src/stores/items.ts`

The `toggleCheck` action has two paths depending on network availability. Both paths call `CacheService.saveItem()` and `syncCounts()` to keep the local state fully consistent.

### Online Path

```
itemService.toggleCheck(listId, itemId)
        │
        ▼ updated Item from server
Replace items[idx] in itemsByList[listId]
        │
        ▼
CacheService.saveItem(updated)     ← write-through to IndexedDB
        │
        ▼
syncCounts(listId)                 ← update checkedCount on list card
```

```typescript
async function toggleCheck(listId: string, itemId: string): Promise<void> {
  try {
    const updated = await itemService.toggleCheck(listId, itemId)
    const items = itemsByList.value[listId] ?? []
    const idx = items.findIndex(i => i.id === itemId)
    if (idx !== -1) items[idx] = updated
    await CacheService.saveItem(updated)
    syncCounts(listId)
  } catch (e) { /* see offline path */ }
}
```

### Offline Path

When `isNetworkError(e)` is true, the toggle is applied locally and enqueued as a `ITEM_CHECK` CRDT operation.

```
Network error detected (no response from axios)
        │
        ▼
Flip checked locally: { ...items[idx], checked: !items[idx].checked }
        │
        ▼
CacheService.saveItem(toggled)     ← IndexedDB updated immediately
        │
        ▼
syncCounts(listId)
        │
        ▼
LocalClockService.getNextClock(listId, deviceId)
        │
        ▼
OperationQueue.enqueue({
  operationType: 'ITEM_CHECK',
  payload: { itemId, checked: toggled.checked, timestamp: Date.now() }
})
```

## CacheService

**File:** `src/services/cache.ts`

`CacheService.saveItem` upserts a single item into the `items` table of the `listme-cache` IndexedDB database, adding the internal `_savedAt` timestamp for ordering.

```typescript
async saveItem(item: Item): Promise<void> {
  await cacheDb.items.put({ ...item, _savedAt: Date.now() })
}
```

On app start, `fetchAll` serves cached items first so the checked state is visible before the network responds:

```typescript
const cached = await CacheService.getItems(listId)
if (cached.length > 0) {
  itemsByList.value[listId] = cached   // ← instant, from IndexedDB
  loading.value = false
}
// then fetches fresh data from server in the background
```

## OperationQueue (Offline)

**File:** `src/crdt/OperationQueue.ts` · `src/crdt/types.ts`

Offline toggles are stored as `CrdtOperation` rows in the `listme-opqueue` IndexedDB database with `_synced: 0`.

```typescript
export interface CrdtOperation {
  id: string                          // client UUID
  listId: string
  deviceId: string
  operationType: OperationType        // 'ITEM_CHECK' for this feature
  payload: Record<string, unknown>    // { itemId, checked, timestamp }
  vectorClock: VectorClockMap
  createdAt: number
}
```

The queue is flushed via `useSyncQueue` when the device comes back online. `OperationQueue.markSynced(opId)` flips `_synced` to `1` after successful replay, and `pruneOld()` removes entries older than 7 days.

## `syncCounts` Helper

After every toggle (online or offline), the list-level counters are recomputed from local state to keep the overview card accurate without a full refetch:

```typescript
function syncCounts(listId: string) {
  const items = itemsByList.value[listId] ?? []
  useListsStore().patchCounts(
    listId,
    items.length,
    items.filter(i => i.checked).length   // checkedCount
  )
}
```

## Acceptance Criteria — Mapping

| Acceptance Criterion | Implementation |
|----------------------|----------------|
| Status nach Neustart erhalten | `CacheService.saveItem()` on every toggle; `fetchAll` loads cache first |
| Offline-Toggle möglich | Local optimistic flip + `OperationQueue.enqueue('ITEM_CHECK', …)` |
| Offline-Ops werden syncronisiert | `useSyncQueue` replays queue on reconnect; `markSynced` marks ops done |
| checkedCount bleibt korrekt | `syncCounts()` recomputes from local array after every toggle |
