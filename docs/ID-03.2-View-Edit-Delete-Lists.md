# ID-03.2 ÔÇö View, edit, delete lists

Issue: ID 3.2
Files:
- Backend: `listme-backend/src/main/java/com/oliwier/listmebackend/api/ListController.java` (GET/PUT/DELETE /api/lists)
- Frontend Store: `listme-frontend/src/stores/lists.ts` (fetchAll, update, remove)
- Frontend UI: `listme-frontend/src/views/HomeView.vue`, `src/components/list/ListCard.vue`, `src/views/ListDetailView.vue`

---

## Overview

This sub-feature covers listing all lists, editing (e.g., name/emoji), and deleting lists. Listing is offline-capable through the local cache. Edit and delete update both the Pinia store and the local cache; deleting also removes related items locally.

---

## View (GET)

Backend:
- Endpoint: GET `/api/lists` Ôćĺ `ListController#getMyLists`
  - Returns all lists in which the current device participates

Frontend:
- Store method: `fetchAll()`
  - First loads from IndexedDB (`CacheService.getLists()`) to render the UI immediately
  - Then fetches fresh data via `listService.getAll()` and writes it back to the cache
- UI: `HomeView.vue`
  - Calls `onMounted(() => listsStore.fetchAll())`
  - Renders `ListCard` components for each list and shows counters/statistics

---

## Edit (PUT)

Backend:
- Endpoint: PUT `/api/lists/{listId}` Ôćĺ `ListController#update`
  - Updates `name` (required) and optionally `emoji`
  - Validates access via `requireAccess`

Frontend:
- Store method: `update(listId, req)`
  - `const updated = await listService.update(listId, req)`
  - Replaces the list entry in the store array and persists via `CacheService.saveList(updated)`
- UI: `ListDetailView.vue`
  - Provides the rename/emoji-change flow (see also document ID-03.3)

Edge cases:
- Invalid name Ôćĺ server validation returns an error
- Missing access Ôćĺ 403 returned by backend

---

## Delete (DELETE)

Backend:
- Endpoint: DELETE `/api/lists/{listId}` Ôćĺ `ListController#delete`
  - Removes the current device's participation; deletes the list only if no participants remain

Frontend:
- Store method: `remove(listId)`
  - `await listService.delete(listId)`
  - Removes the list in the store (`filter`) and calls `CacheService.removeList(listId)`, which also deletes all items of the list from IndexedDB
- UI: `ListCard.vue`
  - Context menu "Delete" calls `listsStore.remove(list.id)`

Edge cases:
- Shared lists: after deleting your own participation, the list remains for other devices
- Offline: delete requires network; UI should show an error or block the action

## Manual verification
1. View:
   - Open app Ôćĺ Home Ôćĺ Lists render immediately from cache, then refresh with API data shortly after
2. Edit:
   - In detail view, change name/emoji Ôćĺ Save
   - Expected: Immediate update of the card and persisted in cache; reload shows new values
3. Delete:
   - On Home, delete a list via the context menu
   - Expected: Card disappears; in IndexedDB the list+items are removed; reloading no longer shows the list
