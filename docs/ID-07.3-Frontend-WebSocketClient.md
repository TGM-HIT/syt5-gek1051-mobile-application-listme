# ID-07.3 — Frontend WebSocket Client & ConnectionBanner

**Issue:** #20
**Files:** `listme-frontend/src/services/websocket.ts`, `listme-frontend/src/stores/presence.ts`, `listme-frontend/src/components/common/ConnectionBanner.vue`

---

## Overview

The frontend WebSocket layer wraps `@stomp/stompjs` in a thin module-level service (`websocket.ts`), keeps connection state outside of Vue's reactivity system for performance, and exposes a minimal API consumed by `useListSync`.

`ConnectionBanner` is the only UI surface — a small floating snackbar that reflects the current connection state to the user.

---

## websocket.ts

**File:** `src/services/websocket.ts`

A plain TypeScript module (not a Vue composable or Pinia store) that holds a single STOMP `Client` instance as module-level state. This avoids duplicating connections across component re-renders.

### Connection URL

```typescript
function buildBrokerUrl(deviceId: string): string {
  const protocol = location.protocol === 'https:' ? 'wss' : 'ws'
  return `${protocol}://${location.host}/ws/websocket?deviceId=${deviceId}`
}
```

The `deviceId` is appended as a query parameter so `DeviceHandshakeInterceptor` on the backend can extract it during the WebSocket handshake (see [ID-07.2](ID-07.2-PresenceHandler-DeviceInterceptor.md)).

The path `/ws/websocket` is the SockJS raw WebSocket endpoint (SockJS appends `/websocket` to the base path `/ws`).

### API

#### `connectWebSocket(): Promise<void>`

Idempotent — returns immediately if already connected. Creates a new `Client`, configures it, calls `activate()`, and resolves the promise on the first `onConnect` callback.

```typescript
// Safe to call multiple times
await connectWebSocket()
```

The `reconnectDelay: 1000` option on the STOMP `Client` enables automatic reconnection with a 1-second base delay. See [ID-07.4](ID-07.4-ReconnectionManager.md) for details.

#### `disconnectWebSocket(): void`

Calls `client.deactivate()` and nulls the client reference. Called when the last list view unmounts.

#### `isConnected(): boolean`

Returns `client?.connected ?? false`. Used by `useListSync` to set its `connected` ref after `connectWebSocket()` resolves.

#### `subscribe(topic, callback): () => void`

Subscribes to a STOMP topic. The callback receives the parsed JSON body (or the raw string if parsing fails). Returns an unsubscribe function — call it to clean up when the component unmounts.

```typescript
const unsub = subscribe(`/topic/list/${listId}`, (payload) => {
  // handle incoming CRDT op
})
// later:
unsub()
```

#### `send(destination, body): void`

Publishes a STOMP SEND frame. No-ops silently if not connected.

```typescript
send(`/app/list/${listId}/join`)
send(`/app/list/${listId}/leave`)
```

## presence.ts (Pinia store)

**File:** `src/stores/presence.ts`

Tracks online devices per list. Keyed by `listId` → `Set<deviceId>`.

```typescript
const onlineByList = ref<Record<string, Set<string>>>({})
```

### API

| Method | Description |
|--------|-------------|
| `setSnapshot(listId, devices)` | Replaces the set for a list (used on `"snapshot"` event) |
| `addDevice(listId, deviceId)` | Adds a device to the set (used on `"joined"` event) |
| `removeDevice(listId, deviceId)` | Removes a device (used on `"left"` event) |
| `getCount(listId)` | Returns `Set.size` — used by `ListDetailView` to display participant count |
| `isOnline(listId, deviceId)` | Returns `true` if a specific device is currently in the set |

Using a `Set` per list (rather than an array) ensures O(1) add/remove and no duplicate entries even if join events arrive twice.

## ConnectionBanner.vue

**File:** `src/components/common/ConnectionBanner.vue`

A small floating pill at the bottom of the screen that shows the current sync status. Used inside `ListDetailView`.

### Props

```typescript
defineProps<{ connected: boolean }>()
```

`connected` comes from `useListSync`'s `connected` ref, which is `true` when the STOMP connection is established.

### Three states

| State | Condition | Color | Message |
|-------|-----------|-------|---------|
| `offline` | `!isOnline` (no network) | Red | "Kein Internet — Offline gespeichert" |
| `syncing` | `isOnline && !connected` | Yellow (pulsing) | "Verbindung wird hergestellt…" |
| `connected` | `isOnline && connected` | Green | "Verbunden" |

`isOnline` is read from the `useOffline` composable, which watches `window.online/offline` events. This means the banner correctly shows "Kein Internet" even if the STOMP client hasn't had a chance to time out yet.

### Visibility logic

- Banner shows immediately on mount if not yet connected.
- Transitions to visible on any status change.
- Auto-hides 2 seconds after transitioning to `connected` (so the user sees confirmation then it disappears).
- Uses `<Teleport to="body">` to escape the `ListDetailView` stacking context.

### Animation

```
enter: slide up 16px + fade in   (cubic-bezier ease)
leave: slide down 16px + fade out
```
