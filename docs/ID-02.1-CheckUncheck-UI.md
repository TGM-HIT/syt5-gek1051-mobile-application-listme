## Overview

This sub-ticket covers the purely visual and interaction layer of the check/uncheck feature. Every item row in the list detail view gets a circular toggle button. Tapping it emits an event upward — the component itself is stateless and carries no store dependency. The `ListDetailView` wires the emitted event to the Pinia store action. Checked and unchecked items are rendered in two separate sections separated by a labelled divider.

## Component: `ItemRow.vue`

**File:** `src/components/item/ItemRow.vue`

`ItemRow` is a purely presentational component — it receives one `Item` prop and emits upward. It never reads from or writes to the store directly.

### Props & Emits

```typescript
const props = defineProps<{ item: Item }>()

const emit = defineEmits<{
  toggle: [itemId: string]   // fired when the checkbox button is tapped
  edit:   [item: Item]
  delete: [itemId: string]
}>()
```

### Checkbox Button

The toggle button uses a filled green circle with a checkmark SVG when `item.checked` is `true`, and an empty bordered circle otherwise. Both states have `transition-all duration-200` for smooth animation.

```html
<button
  @click="emit('toggle', item.id)"
  :aria-label="item.checked ? 'Als offen markieren' : 'Artikel erledigen'"
  class="flex-shrink-0 w-6 h-6 rounded-full border-2 transition-all duration-200
         flex items-center justify-center"
  :class="item.checked
    ? 'bg-ctp-green border-ctp-green'
    : 'border-ctp-surface2 hover:border-ctp-green'"
>
  <svg v-if="item.checked" class="w-3.5 h-3.5 text-ctp-base" fill="none"
       viewBox="0 0 24 24" stroke="currentColor" stroke-width="3">
    <path stroke-linecap="round" stroke-linejoin="round" d="M5 13l4 4L19 7" />
  </svg>
</button>
```

### Visual Feedback

Two CSS classes are applied conditionally on the row wrapper and the item name span:

| State | Row wrapper | Name `<span>` |
|-------|-------------|---------------|
| `checked: false` | no extra class (full opacity) | `text-ctp-text` — normal |
| `checked: true` | `opacity-50` | `line-through text-ctp-overlay1` |

```html
<!-- Row wrapper -->
<div :class="item.checked ? 'opacity-50' : ''">

<!-- Name span -->
<span
  class="text-sm font-medium text-ctp-text truncate block transition-all duration-200"
  :class="item.checked ? 'line-through text-ctp-overlay1' : ''"
>
  {{ item.name }}
</span>
```

## View: `ListDetailView.vue`

**File:** `src/views/ListDetailView.vue`

The list detail view splits items into two computed lists and renders them with a divider. It handles the `toggle` event from `ItemRow` and delegates to the store.

### Item Splitting

```typescript
const filteredUncheckedItems = computed(() =>
  filteredItems.value.filter(i => !i.checked)
)
const filteredCheckedItems = computed(() =>
  filteredItems.value.filter(i => i.checked)
)
```

### Toggle Handler

```typescript
async function onToggle(lid: string, itemId: string) {
  await itemsStore.toggleCheck(lid, itemId)
  itemsVersion.value++   // bumps BudgetBar recomputation
}
```

### Rendering

```html
<!-- Unchecked items -->
<div class="group" v-for="item in filteredUncheckedItems" :key="item.id">
  <ItemRow :item="item" @toggle="onToggle(listId, $event)" … />
</div>

<!-- Divider — only shown when both sections are non-empty -->
<div v-if="filteredUncheckedItems.length > 0 && filteredCheckedItems.length > 0"
     class="flex items-center gap-3 py-2 px-4">
  <div class="flex-1 h-px bg-ctp-surface1" />
  <span class="text-xs text-ctp-overlay0">Erledigt</span>
  <div class="flex-1 h-px bg-ctp-surface1" />
</div>

<!-- Checked items -->
<div class="group" v-for="item in filteredCheckedItems" :key="item.id">
  <ItemRow :item="item" @toggle="onToggle(listId, $event)" … />
</div>
```

### Progress Chip & Bar

The header shows a `checkedCount / itemCount` chip and a progress bar that updates reactively whenever `list.checkedCount` changes (kept in sync by `syncCounts` in the store).

```typescript
const progressPct = computed(() => {
  if (!list.value || list.value.itemCount === 0) return 0
  return Math.round((list.value.checkedCount / list.value.itemCount) * 100)
})
```
