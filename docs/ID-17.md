# ID-17 — Geräteübergreifende Synchronisierung via Sync-Link

**User Story:** Als Benutzer möchte ich mich über einen eindeutigen, meinem Konto zugeordneten Link authentifizieren können, damit ich kein Passwort erstellen oder eingeben muss.

**Priorität:** Should Have | **SP:** 5 | **HEAD:** PRZO

---

## Overview

Because ListMe is account-free, "authentication" means linking all of a user's devices under one identity. The user generates a sync link on Device A; opening it on Device B pulls down all lists and establishes the same `userId`, so both devices share data going forward.

---

## Backend

- `sync_tokens` table: `token` (PK, 24-char random), `created_by_device`, `user_id`
- `sync_token_lists` table: maps tokens to list IDs
- `SyncTokenController`
  - `POST /api/sync-tokens` — generates a token grouping all lists of the current device; returns `{ token, url }`
  - `GET /api/sync-tokens/{token}` — resolves the token, returns the list of list IDs and the `userId` to adopt

---

## Frontend

- `src/views/SyncApplyView.vue` — route `/sync/:token`; on mount calls the resolve endpoint, stores the returned `userId` in `localStorage`, and pulls down all linked lists
- `src/components/list/LinkDevicesModal.vue` — generates the sync link and displays it as a copyable URL and QR code
- Settings screen → "Geräte verknüpfen" → opens `LinkDevicesModal`

---

## Manual Verification

1. On Device A → Settings → "Geräte verknüpfen" → copy the link.
2. Open the link on Device B → verify all of Device A's lists appear on Device B.
3. Add an item on Device B → verify it syncs back to Device A in real time.
4. Verify the token is single-use or expires after a reasonable time.
