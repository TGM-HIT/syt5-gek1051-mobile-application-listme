# ID-04.1 — Categories (examples: Fruit, Vegetables, Dairy, Other)

Issue: ID 4.1

Files to know:
- Backend API: `listme-backend/src/main/java/com/oliwier/listmebackend/api/CategoryController.java`
- Backend DTOs: `.../api/dto/CreateCategoryRequest.java`, `.../api/dto/CategoryResponse.java`
- DB migration: `listme-backend/src/main/resources/db/migration/V3__labels_categories.sql` (adds categories and item.category_id)
- Frontend types: `listme-frontend/src/types/index.ts` (Category, Item.categoryId/categoryName/categoryColor)
- Frontend UI: category chip on each item row → `listme-frontend/src/components/item/ItemRow.vue`

---

## Overview

Items can belong to a category (per list). Typical examples are:
- Fruit
- Vegetables
- Dairy
- Other

The app supports per-list categories in the backend. Each Item optionally references a Category via `categoryId`. The UI displays the category next to the item name as a small chip. Colors can be used to visually distinguish categories when provided (`categoryColor`).

---

## Backend

- List categories of a list:
  - GET `/api/lists/{listId}/categories`
- Create a category:
  - POST `/api/lists/{listId}/categories` with body `{ name: string, color?: string }` (see `CreateCategoryRequest`)
- Update a category:
  - PUT `/api/lists/{listId}/categories/{categoryId}` with the same body shape
- Delete a category:
  - DELETE `/api/lists/{listId}/categories/{categoryId}`

Notes:
- Access is scoped to list participants (see `CategoryController` + `CategoryService`).
- Deleting a category sets `item.category_id` to NULL (due to `ON DELETE SET NULL`), so items are preserved.

---

## Frontend

- The `Item` type has the fields: `categoryId`, `categoryName`, `categoryColor` (all nullable).
- The row component `ItemRow.vue` renders a category chip when `item.categoryName` is present.
- Category management UI (listing/creating/updating categories) follows the standard pattern but may be surfaced from an Add/Edit Item sheet.

---

## Defaults and i18n

- If no category is chosen, `categoryId = null` and no chip is displayed.
- Example preset categories in product copy: “Fruit, Vegetables, Dairy, Other.”
- Actual display names are user-entered or localized via the UI; there is no enforced global vocabulary.

---

## Manual verification

1) Fetch categories for a list
- Call GET `/api/lists/{listId}/categories`
- Expect an array of categories (empty array if none)

2) Create a category
- POST with `{ name: "Dairy", color: "#00AEEF" }`
- Expect 201 Created with the new category

3) Delete the category
- DELETE the created ID
- Expect 204 No Content; items previously assigned to it should remain but show without a category chip
