# ID-03.3 — Rename lists

Issue: ID 3.3
Files:
- Backend: `listme-backend/src/main/java/com/oliwier/listmebackend/api/ListController.java` (PUT /api/lists/{listId})
- Frontend Store: `listme-frontend/src/stores/lists.ts` (update)
- Frontend UI: `listme-frontend/src/views/ListDetailView.vue` (settings/edit area)

---

## Overview

As a user, I want to rename existing shopping lists. Optionally, the emoji can also be changed. The change is persisted via the API and simultaneously saved locally in the cache so that a reload shows the new data.

---

## API design (backend)

Endpoint: PUT `/api/lists/{listId}`
- See `ListController#update`
- Expects `UpdateListRequest { name: string; emoji?: string }`
- Sets `name` and optionally `emoji`; validates access (the device must participate in the list)
- Response: `ListResponse` with updated fields

---

## Frontend behavior

Store: `useListsStore.update(listId, req)`
- Calls `listService.update(listId, req)`
- Replaces the list in the store array (`lists[idx] = updated`)
- Persists in IndexedDB (`CacheService.saveList(updated)`) 

UI: `ListDetailView.vue`
- Provides the rename flow (e.g., inline edit or dialog)
- Uses `listsStore.getById(listId)` for display and `listsStore.update(...)` to save
- Updates dependent UI (title, cards, breadcrumbs)

---

## Errors and edge cases
- Empty/too short name → server validation returns an error; UI should display a hint
- No access (not a participant) → 403 Forbidden
- Offline: renaming requires network; UI should block saving or show a clear error

---

## Manual verification
1. Open a list → start renaming → enter a new name → Save
2. Expected: Title updates immediately; after reload it remains updated
3. Also change the emoji → Save
4. Expected: Card/detail shows the new emoji; store and cache hold updated values
