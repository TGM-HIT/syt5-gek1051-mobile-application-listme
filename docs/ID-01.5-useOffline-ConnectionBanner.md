# ID-01.5 — useOffline Composable & ConnectionBanner

**Issue:** #27
**Files:**
- `listme-frontend/src/composables/useOffline.ts` — reactive online state singleton
- `listme-frontend/src/components/common/ConnectionBanner.vue` — UI feedback component

---

## Overview

These two pieces work together to give the user clear, real-time feedback about their network and sync status. `useOffline` is the source of truth for whether the device has internet. `ConnectionBanner` turns that state (plus the WebSocket connection state) into a visible snackbar at the bottom of the screen.

---

## useOffline

**File:** `src/composables/useOffline.ts`

### What it does

Exposes a single reactive `isOnline` ref that mirrors `navigator.onLine` and updates in real time whenever the device goes offline or comes back online.

### Implementation

```typescript
// Module-level (singleton) — runs once when the module is first imported
const isOnline = ref(navigator.onLine)

window.addEventListener('online',  () => { isOnline.value = true  })
window.addEventListener('offline', () => { isOnline.value = false })

export function useOffline() {
  return { isOnline }
}
```

### Singleton pattern

The `isOnline` ref and the event listeners live at **module level**, outside the `useOffline` function. This means:

- Only **one pair of event listeners** is ever registered, regardless of how many components call `useOffline()`
- All consumers (ConnectionBanner, useSyncQueue, Pinia stores) share the **exact same ref** — when it changes, every watcher reacts simultaneously
- No cleanup needed — the listeners persist for the app's lifetime

### Usage

```typescript
import { useOffline } from '../composables/useOffline'

const { isOnline } = useOffline()

// Reactive — updates automatically
watch(isOnline, (online) => {
  if (online) console.log('Back online!')
})

// Or use directly in template
// v-if="!isOnline" → show offline message
```

### SSR / non-browser safety

The implementation guards against non-browser environments:

```typescript
const isOnline = ref(typeof navigator !== 'undefined' ? navigator.onLine : true)

if (typeof window !== 'undefined') {
  window.addEventListener(...)
}
```

This prevents crashes during server-side rendering or unit tests where `window` doesn't exist.

## ConnectionBanner

**File:** `src/components/common/ConnectionBanner.vue`

### What it does

A floating pill-shaped snackbar that appears at the bottom of the screen to communicate three states:

| State | Condition | Color | Message |
|-------|-----------|-------|---------|
| `offline` | `!isOnline` | Red (`ctp-red`) | `Kein Internet — Offline gespeichert` |
| `syncing` | `isOnline && !connected` | Yellow (`ctp-yellow`) | `Verbindung wird hergestellt…` |
| `connected` | `isOnline && connected` | Green (`ctp-green`) | `Verbunden` |

The `connected` prop is the WebSocket connection state passed in from `ListDetailView` (via `useListSync`). So there are two layers:
- **Network level** — is the device online at all? (`isOnline`)
- **WebSocket level** — is the STOMP connection to the server established? (`connected` prop)

### Props

| Prop | Type | Description |
|------|------|-------------|
| `connected` | `boolean` | Whether the WebSocket/STOMP connection is established for the current list |

### Visibility logic

The banner is not always visible — showing it permanently for "connected" would be noise:

```
onMounted:
    if status !== 'connected' → show immediately
    (so the user sees the offline/syncing state on page load if relevant)

watch(status):
    always show on any state change
    if status === 'connected' (and was previously something else):
        auto-hide after 2 seconds
    if status → offline or syncing:
        stays visible until it changes again
```

### Teleport

The banner uses `<Teleport to="body">` to render outside the normal component tree. This prevents it from being clipped by parent elements with `overflow: hidden` or `z-index` stacking contexts (e.g. list cards, bottom sheets).

### Animation

```css
.snackbar-enter-active,
.snackbar-leave-active {
  transition: all 0.3s cubic-bezier(0.32, 0.72, 0, 1);
}
.snackbar-enter-from,
.snackbar-leave-to {
  opacity: 0;
  transform: translateX(-50%) translateY(16px);  /* slides up from below */
}
```

The banner slides up from below the screen when it appears and slides back down when it hides. The `translateX(-50%)` maintains the horizontal centering (`left-1/2 -translate-x-1/2`) during the animation.

### Usage

```vue
<!-- In ListDetailView.vue -->
<ConnectionBanner :connected="wsConnected" />

<!-- wsConnected comes from useListSync() -->
const { isConnected: wsConnected } = useListSync(listId)
```

```
                      ┌─────────────────────────────────┐
                      │          OFFLINE                │
                      │  (isOnline = false)             │
                      │  Banner: red, always visible    │
                      └────────────────┬────────────────┘
                                       │ network restored
                                       ▼
                      ┌─────────────────────────────────┐
                      │          SYNCING                │
                      │  (isOnline = true,              │
                      │   connected = false)            │
                      │  Banner: yellow, pulse dot      │
                      └────────────────┬────────────────┘
                                       │ WebSocket established
                                       ▼
                      ┌─────────────────────────────────┐
                      │          CONNECTED              │
                      │  (isOnline = true,              │
                      │   connected = true)             │
                      │  Banner: green, hides after 2s  │
                      └─────────────────────────────────┘
```

## Where useSyncQueue fits in

`useOffline.isOnline` is also watched by `useSyncQueue` (see [ID-01.3-SyncQueue.md](ID-01.3-SyncQueue.md)). The transition from `offline → online` triggers two things simultaneously:

1. `ConnectionBanner` transitions from red → yellow
2. `useSyncQueue` starts flushing pending operations

Once the flush completes and the WebSocket reconnects, `ConnectionBanner` transitions from yellow → green and auto-hides.
