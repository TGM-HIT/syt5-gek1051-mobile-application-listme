# ID-13 — Listen duplizieren

**User Story:** Als Benutzer möchte ich Einkaufslisten duplizieren können, damit ich wiederkehrende Einkäufe schneller anlegen kann.

**Priorität:** Should Have | **SP:** 3 | **HEAD:** HIRA

---

## Overview

A user can duplicate any list they own. The duplicate gets a new UUID, the same name with a copy suffix, and all items copied (unchecked). The duplicate is private to the duplicating device — it is not shared with the original list's participants.

---

## Backend

- `ListService.duplicate(listId, device)` — clones `ShoppingList` and all non-deleted `Item` rows; resets `checked = false`, generates new UUIDs
- `ListController` — `POST /api/lists/{listId}/duplicate` → returns the new `ListResponse`

---

## Frontend

- List detail header menu or list card context menu → "Duplizieren" option
- Calls `listService.duplicate(listId)` → prepends the new list to the store
- Navigates the user to the new list detail view

---

## Manual Verification

1. Open a list with several items → tap "Duplizieren" from the menu.
2. Verify a new list appears on the home screen with the same name and `(Kopie)` suffix.
3. Open the duplicate → verify all items are present and unchecked.
4. Verify the original list is unchanged.
