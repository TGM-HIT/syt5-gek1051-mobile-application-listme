# ID-04.3 — Search across all items

Issue: ID 4.3

Files to know:
- Backend API: `listme-backend/src/main/java/com/oliwier/listmebackend/api/ItemController.java` (`getItems` supports `?q=`)
- Backend Service: `.../domain/service/ItemService.java` (filters by query)
- Frontend view: `listme-frontend/src/views/ListDetailView.vue` (search input, filtered list)
- Frontend store: `listme-frontend/src/stores/items.ts` (provides items, cache-first)
- Frontend service: `listme-frontend/src/services/item.ts` (GET /items)

---

## Overview

Users can quickly find items by typing in the search field on the list detail screen. The search considers item names (and may include labels in backend logic). The UI narrows the visible list as the user types.

---

## Backend

- Endpoint: `GET /api/lists/{listId}/items?q={query}`
- When `q` is provided, the backend returns only items matching the search term.
- When omitted, all active items are returned.

Note: The UI also applies a client-side filter to the already-fetched items so results appear instantly. The backend `q` parameter is useful for initial loads, long lists, or when directly linking to a search.

---

## Frontend behavior

- On mount, `itemsStore.fetchAll(listId)` loads items cache-first and then from the API.
- As the user types into the search field, a computed `filteredItems` narrows the current items array to matches (case-insensitive substring).
- Debounce: The field should debounce network-backed searches; however, the current implementation focuses on in-memory filtering for responsiveness. If needed, wire a watcher to call `itemService.getAll(listId, q)` (not required today).

---

## Offline support

- Because the store hydrates from IndexedDB via `CacheService.getItems(listId)`, search works offline against cached data.
- No Service Worker caching for `/api/**` is used; dynamic data lives in IndexedDB (see ID-01.2 and ID-01.4).

---

## Manual verification

1) Type a substring present in multiple items
- Expect the visible list to reduce to matching items immediately.

2) Clear the search
- Expect the full list to be shown again.

3) Offline
- Reload the app offline and search for a previously cached item name.
- Expect the filter to work against cached data.
